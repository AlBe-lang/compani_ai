# Part0 개발 일지 (2) — Stage 2: 로컬 실행 환경 + 테스트 체계
> 작성일: 2026-02-21 | Part: 0 | Stage: 2

---

## 1. 작업 내용
Stage 2에서는 로컬 실행/검증 표준화를 위해 `deploy/setup.sh`, `Makefile`, `pyproject.toml`, `.flake8`, `mypy.ini`, `.gitignore`를 구성했습니다.  
테스트 구조를 `tests/unit`, `tests/integration`, `tests/e2e`로 분리하고 `tests/conftest.py`에 공통 Mock fixture를 추가했습니다.  
코드 리뷰 피드백을 반영해 logger fallback 보강, `sys.path` 조작 제거, Mock 타입 강화, 테스트 중복 제거 및 로거 메서드(`debug/warning/error`) 검증까지 반영했습니다.

## 2. 결과물
생성된 파일: `deploy/setup.sh`, `Makefile`, `.flake8`, `mypy.ini`, `tests/conftest.py`, `tests/unit/test_error_codes.py`, `tests/unit/test_ids.py`, `tests/unit/test_logger.py` — 실행/테스트 기반.  
수정된 파일: `pyproject.toml` — pytest pythonpath/asyncio 설정, black/isort 설정 추가.  
수정된 파일: `src/observability/logger.py` — fallback logger 레벨 메서드 보강, `sys.stdout.write` 출력, structlog 캐시 동작 수정.  
수정된 파일: `.gitignore`, `tests/unit/test_stage1_smoke.py`, `tests/unit/test_ids.py`, `tests/unit/test_logger.py`, `mypy.ini`, `deploy/setup.sh` — 리뷰 이슈 해소.

## 3. 결정 이유
`pythonpath=["src"]`를 `pyproject.toml`에 고정해 테스트 파일의 경로 해킹(`sys.path.insert`)을 제거하고 CI/로컬 일관성을 확보했습니다.  
테스트는 스모크와 전용 단위 테스트의 역할을 분리해 중복을 줄이고, 회귀 포인트를 각 테스트 파일에서 명확히 추적할 수 있게 했습니다.  
전략 문서(개발 일지(4).md) Part 0 Stage 2와 정렬되도록 실행 명령/테스트 디렉토리/공통 fixture를 우선 완료했으며, 전략 변경 이력은 없음입니다.

## 4. 발생한 리스크
리스크: 로컬 환경에 `black/flake8/isort` 미설치로 `make lint` 완전 통과 확인 불가 | 심각도: B | 발현 시점: Stage 2 Gate 점검 시 | 대응: `python -m pip install -r requirements-dev.txt` 후 lint 재실행.  
리스크: `make` 명령이 없는 Windows 환경에서 Makefile 타겟 실행이 제한될 수 있음 | 심각도: C | 발현 시점: 로컬 개발 중 | 대응: PowerShell 대체 스크립트 또는 `python -m ...` 직접 실행 경로 병행.
