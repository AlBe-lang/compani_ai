## 1. 작업 내용
리뷰 권고 순서(A → B → C)대로 CI 흐름, 도메인 계약, 포트 시그니처, 테스트 품질을 보강했습니다.

도메인 계약에서는 `AgentRole` Enum 도입, `WorkItem.result`를 `TaskResult` 타입으로 정규화, `ErrorCode`를 `domain/contracts`로 이동해 의존성 방향을 정리했습니다.

## 3. 결정 이유
`ErrorCode`를 도메인 계약으로 이동한 이유는 `TaskResult.error_code`를 규칙(F-07)에 맞춰 Enum으로 강제하면서도 `domain -> observability` 역의존을 피하기 위함입니다.  
`MessageQueuePort.send(..., context=...)`는 메시지 계약의 `context` 필드와 구현체 시그니처를 정렬해 계약 불일치를 제거했습니다.  
전략 문서 기준 Stage 완료 이후의 품질 보강 작업이며, 변경 이력은 BREAKING CHANGE 성격(도메인 계약 타입 강화)을 포함합니다.
