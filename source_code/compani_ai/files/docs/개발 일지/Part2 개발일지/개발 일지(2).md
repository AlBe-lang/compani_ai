# Part2 개발 일지 (2) — Stage 1 리뷰 반영 패치
> 작성일: 2026-02-23 | Part: 2 | Stage: 1 보강

---

## 1. 작업 내용
PM 코드 리뷰 지적사항 중 우선순위가 높은 항목을 반영했습니다.  
`B-level` 이슈였던 필수 로그 이벤트 3개(`slm.deps.waiting`, `slm.qa.sent`, `slm.llm.call`)를 실행 흐름에 삽입했습니다.  
`C-level` 이슈 2건도 함께 처리해 fallback 프롬프트 하드코딩 제거와 Stage 1 프롬프트 보안 기본 지시를 추가했습니다.

## 2. 결과물
수정 파일:
- `src/application/base_agent.py`
  - 의존성 대기 진입 로그: `slm.deps.waiting`
  - Q&A 전송 로그: `slm.qa.sent`
  - LLM 호출 로그: `slm.llm.call`
- `src/application/backend_agent.py`
  - 프롬프트 파일 누락 시 문자열 fallback 대신 `E_SYSTEM_CONFIG` 예외 발생
- `src/application/prompts/backend/stage1_basic.txt`
  - 보안 기본 규칙(입력 검증, raw SQL 문자열 결합 금지) 추가
- `tests/unit/test_backend_stage1.py`
  - 필수 실행 로그 3개 출력 검증 테스트 추가

## 3. 결정 이유
`B-level` 이슈는 Stage 2 착수 전 선반영이 권장된 항목이라 즉시 반영했습니다.  
로그 이벤트 3개는 운영 관측성 최소 기준이며, 로직 변경 없이 이벤트만 추가할 수 있어 리스크가 낮습니다.  
`C-level` 2건도 영향 범위가 작아 함께 정리하는 편이 이후 Stage에서 기술 부채 누적을 줄입니다.

## 4. 발생한 리스크
