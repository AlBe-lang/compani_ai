## 1. 작업 내용
Part 2 Stage 2 범위만 고정해 Backend 에이전트의 stage config 전환을 추가했습니다.  
`BackendSLMConfig.stage` 값에 따라 프롬프트를 `stage1_basic.txt` 또는 `stage2_database.txt`로 선택하도록 구현했습니다.  
Stage 2에서는 `super()._validate_files()`를 유지한 상태로 경로 패턴 4개(`database.py`, `models`, `alembic/env.py`, `alembic/versions/*.py`)를 추가 검증하도록 확장했습니다.

## 3. 결정 이유
Stage는 역할 분리가 아니라 동일 에이전트의 능력 레벨 확장이므로 config 분기 방식을 선택했습니다.  
검증은 고정 파일명 대신 경로 패턴 기반으로 설정해 LLM의 파일명 편차(예: `0001_initial.py`)를 허용했습니다.  
개발 일지 번호는 Stage 2임에도 `(3)`으로 기록했으며, Stage 1 리뷰 반영 패치가 `개발 일지(2)`를 사용한 이력을 보존하기 위한 결정입니다(D-05 소급 수정 금지).
