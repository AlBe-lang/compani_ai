## 1. 프로젝트 개요

### 1.1 프로젝트명

**실시간 협업 멀티 에이전트 AI 개발 시스템 (Multi-Agent Collaborative Development System)**

### 1.2 한 줄 정의

> CTO 역할의 대형 언어 모델(LLM)이 프로젝트를 기획·분배하고, 역할별로 파인튜닝된 경량 언어 모델(SLM) 에이전트들이 실시간으로 협업하여 소프트웨어를 자동 완성하는 완전 로컬 AI 개발 시스템

### 1.3 핵심 수치

| 항목 | 값 |
|------|----|
| 개발 비용 | $0 (완전 오픈소스, 로컬 실행) |
| 월 운영비 | 약 5,000원 (전기세만) |
| 목표 프로젝트 완성도 | 60–70% (사람이 마무리 가능한 수준) |
| MVP 개발 기간 | 3주 |
| 전체 시스템 개발 기간 | 3–4개월 (8단계) |
| 하드웨어 | Mac Mini M4 16GB (로컬 전용) |

## 2. 문제 정의

### 2.1 현재 상황의 문제점

#### 문제 1: 높은 AI API 비용
- GPT-4 기반 개발 자동화 도구: 프로젝트당 $10–$100 비용 발생
- 대규모 프로젝트 적용 시 연간 수백만 원 이상 지출
- 비용 제약으로 중소기업·스타트업의 AI 활용 제한

#### 문제 2: 보안 위협
- 외부 API 사용 시 코드·설계 정보가 서드파티 서버로 전송됨
- 정부·공공기관·금융사 등 보안 민감 조직은 사실상 사용 불가
- GDPR, 개인정보보호법 등 규정 위반 위험

#### 문제 3: 단일 에이전트의 한계
- 단일 LLM은 Frontend·Backend·DevOps 등 전 영역을 동시에 잘 처리하지 못함
- 역할별 전문성이 부족해 생성 코드의 품질이 불균일
- 에이전트 간 협업·검토 프로세스 부재로 오류 전파

#### 문제 4: 자가 개선 불가
- 기존 도구는 이전 프로젝트 경험을 학습에 활용하지 못함
- 반복되는 실수를 수정하는 메커니즘 없음
- 프로젝트마다 처음부터 동일한 수준으로 시작

### 2.2 기회

- 오픈소스 LLM(Llama, Phi 등)의 빠른 품질 향상
- Ollama를 통한 로컬 LLM 실행 대중화
- 멀티 에이전트 협업 패턴의 연구 성숙
- 로컬 하드웨어(Apple Silicon M4) 성능 대폭 향상

## 3. 솔루션

### 3.1 핵심 아이디어

**"CTO + 전문 개발팀"을 AI 에이전트로 구현**

```
사람 조직             →    AI 에이전트 시스템
─────────────────────────────────────────────
CTO (기획·조율)       →    CTO Agent (Llama 3.1 70B)
Frontend 개발자       →    Frontend SLM (Phi-3.5, 파인튜닝)
Backend 개발자        →    Backend SLM (Phi-3.5, 파인튜닝)
DevOps 엔지니어       →    MLOps SLM (Llama 3.2 3B, 파인튜닝)
팀 회의               →    Stage Gate Meeting 시스템
코드 리뷰             →    피어 리뷰 자동화
성과 기반 성장        →    DNA 진화 시스템
```

### 3.2 동작 방식

```
사용자 입력 (자연어 프로젝트 아이디어)
          │
          ▼
    CTO Agent가 전략 수립 및 작업 분해
          │
    ┌─────┼─────┐
    ▼     ▼     ▼
 Frontend Backend MLOps
 (병렬 실행, 실시간 협업)
    │     │     │
    └─────┼─────┘
          ▼
  Shared Workspace에서 결과 통합
          │
          ▼
  Stage Gate 회의 (집단 지성 검증)
          │
          ▼
  최종 코드 및 파일 생성
```

## 4. 핵심 가치 제안

### 4.1 차별화 포인트

| 항목 | 기존 솔루션 | 본 시스템 |
|------|-----------|----------|
| 비용 | $10–$100/프로젝트 | $0 (완전 로컬) |
| 보안 | 외부 서버 전송 | 완전 폐쇄망 |
| 전문성 | 단일 LLM 범용 처리 | 역할별 파인튜닝 |
| 협업 | 단일 에이전트 | 실시간 다중 에이전트 협업 |
| 학습 | 고정 성능 | DNA 진화로 지속 개선 |
| 검증 | 없음 | Stage Gate 집단 지성 검증 |

### 4.2 6대 핵심 가치

1. **비용 효율** — 오픈소스 모델 + 로컬 실행 = $0 API 비용
2. **완전 보안** — 데이터가 시스템 밖을 나가지 않음
3. **실시간 협업** — 에이전트들이 서로 소통하고 도움을 요청
4. **역할 전문성** — 파인튜닝으로 각 역할에 특화된 능력 보유
5. **집단 지성** — Stage Gate 회의로 단계별 품질 검증
6. **자가 진화** — DNA 시스템으로 프로젝트마다 성능 향상

## 6. 기능 범위

### 6.1 MVP 포함 기능 (Must Have)

| 기능 | 설명 | 우선순위 |
|------|------|---------|
| CTO Agent | 프로젝트 전략 수립, 작업 분해, 에이전트 조율 | P0 |
| Frontend SLM | React/Flutter UI 코드 생성 | P0 |
| Backend SLM | FastAPI/Express API 및 DB 스키마 생성 | P0 |
| MLOps SLM | Dockerfile, docker-compose, CI/CD 설정 | P0 |
| WorkItem 추적 | 작업 상태 관리 (planned → in_progress → done) | P0 |
| 의존성 처리 | 에이전트 간 작업 의존성 대기·알림 | P0 |
| 질문-답변 시스템 | 에이전트 간 정보 요청 및 응답 | P1 |
| 결과물 파일 생성 | 구조화된 코드 파일 및 README 자동 생성 | P0 |

### 6.2 전체 시스템 기능 (Full Scope)

| 기능 | 설명 | 단계 |
|------|------|------|
| 역할별 파인튜닝 | Unsloth LoRA로 각 SLM 특화 학습 | Phase 2 |
| Stage Gate 회의 | 각 단계 완료 후 집단 지성 검증 | Phase 5 |
| DNA 진화 시스템 | 10개 유전자 기반 에이전트 행동 진화 | Phase 6 |
| 긴급 미니 회의 | 블로킹 이슈 감지 시 자동 소집 | Phase 7 |
| 피어 리뷰 | 작업 완료 시 자동 코드 리뷰 | Phase 7 |
| CEO 대시보드 | Flutter Web 기반 실시간 모니터링 UI | Phase 8 |
| 지식 그래프 | 에이전트 전문성 추적, 최적 응답자 매핑 | Phase 4 |

### 6.3 MVP 제외 기능 (Out of Scope)

- 파인튜닝 (Phase 2) — 시간 소요, MVP 후 추가
- Stage Gate Meeting (Phase 5) — 복잡도 높음
- DNA 진화 (Phase 6) — 복수 프로젝트 경험 필요
- 고급 협업 기능 (Phase 7) — 긴급 회의, 피어 리뷰
- CEO 대시보드 (Phase 8) — UI 개발 우선순위 낮음

## 7. MVP 계획 (3주)

### 7.1 일정 개요

```
Week 1 (Day 1–7)   : 핵심 인프라 구축
Week 2 (Day 8–14)  : 협업 시스템 구현
Week 3 (Day 15–21) : 완성 및 검증
```

### 7.2 Week 1 — 핵심 인프라 (Day 1–7)

| Day | 작업 | 산출물 |
|-----|------|--------|
| 1–2 | Ollama 설치, 모델 다운로드, Python 환경 구축 | 실행 환경 완성 |
| 3–4 | BaseAgent 추상 클래스, CTOAgent 구현 | CTO 전략 수립 동작 |
| 5–6 | SimpleSLM (Frontend/Backend/MLOps) 구현 | 역할별 코드 생성 동작 |
| 7   | main.py 통합, 첫 Todo 앱 생성 테스트 | 전체 파이프라인 동작 |

**Week 1 완료 기준**: CTO가 전략 수립 → SLM들이 순차 실행하여 코드 파일 생성

### 7.3 Week 2 — 협업 시스템 (Day 8–14)

| Day | 작업 | 산출물 |
|-----|------|--------|
| 8–9   | WorkItem 클래스, WorkSpace CRUD, 상태 관리 | 작업 상태 추적 |
| 10–11 | 의존성 추적, 대기 메커니즘, 완료 알림 | 에이전트 간 의존성 처리 |
| 12–13 | Message 클래스, 메시지 큐, 질문-답변 라우팅 | Q&A 시스템 |
| 14    | SLM 협업 통합, 전체 테스트, 버그 수정 | 협업 파이프라인 |

**Week 2 완료 기준**: 에이전트들이 서로 질문하고 의존성을 기다리며 협업

### 7.4 Week 3 — 완성 및 검증 (Day 15–21)

| Day | 작업 | 산출물 |
|-----|------|--------|
| 15–16 | 파일 저장 시스템, 프로젝트 구조 생성, README 생성 | 완성된 파일 출력 |
| 17–18 | 3개 테스트 프로젝트 (Todo, 블로그, 방명록) | 검증된 시스템 |
| 19–20 | 에러 핸들링 강화, 로깅, 성능 개선 | 안정화된 시스템 |
| 21    | 데모 시나리오 작성, 데모 실행, 결과 문서화 | MVP 완성 |

**Week 3 완료 기준**: 3개 프로젝트 60%+ 자동 완성

### 7.5 테스트 프로젝트

| 프로젝트 | 복잡도 | 목적 | 예상 시간 |
|---------|--------|------|----------|
| Todo 앱 (CRUD) | 낮음 | 기본 기능 검증 | 30분 |
| 간단한 블로그 | 중간 | 협업 기능 검증 | 1시간 |
| 방명록 앱 | 낮음–중간 | 전체 시스템 안정성 검증 | 45분 |

## 8. 전체 로드맵 (8단계)

```
Phase 1  (1–2주)  : 기본 인프라 ←── MVP 핵심
Phase 2  (2–3주)  : SLM 파인튜닝
Phase 3  (2주)    : Shared Workspace 완성
Phase 4  (2주)    : Communication Hub
Phase 5  (1–2주)  : Stage Gate Meeting
Phase 6  (2주)    : DNA 진화 시스템
Phase 7  (2주)    : 고급 협업 기능
Phase 8  (2–3주)  : 통합 최적화 + CEO 대시보드
─────────────────────────────────────────────
총 기간: 약 3–4개월
```

### 단계별 상세

| Phase | 핵심 목표 | 주요 산출물 |
|-------|----------|-----------|
| Phase 1 | CTO + SLM 기본 동작 | BaseAgent, CTOAgent, SimpleSLM |
| Phase 2 | 역할별 SLM 특화 | 파인튜닝 데이터셋 1,000개 × 3역할, 학습된 모델 |
| Phase 3 | 실시간 작업 공유 | SharedWorkspace, 이벤트 시스템, Redis 연동 |
| Phase 4 | 지능형 에이전트 통신 | CommunicationHub, 지식 그래프, 자동 라우팅 |
| Phase 5 | 집단 지성 검증 | StageGate 회의 로직, 회의록 자동 생성 |
| Phase 6 | 에이전트 자가 진화 | AgentDNA, EvolutionEngine, 진화 히스토리 |
| Phase 7 | 완전 자율 협업 | 긴급 회의, 피어 리뷰, 자동 리뷰어 선정 |
| Phase 8 | 시스템 완성 | 성능 최적화, CEO 대시보드, 실전 프로젝트 3개 검증 |

## 10. 성공 지표 (KPI)

### 10.1 MVP KPI

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 프로젝트 완성도 | ≥ 60% | 수동 평가 (요구사항 대비 완성 항목 비율) |
| 작업 완료율 | ≥ 80% | 완료된 WorkItem / 전체 WorkItem |
| 협업 성공률 | ≥ 70% | 성공한 Q&A 교환 / 전체 Q&A 요청 |
| 시스템 안정성 | ≥ 90% | 에러 없이 완료된 실행 / 전체 실행 |
