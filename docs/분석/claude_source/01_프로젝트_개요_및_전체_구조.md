## 1. 프로젝트 개요

### 1.1 기본 정보

| 항목 | 내용 |
|------|------|
| 프로젝트명 | Claude Code (`@anthropic-ai/claude-code`) |
| 제작사 | Anthropic |
| 소스 공개 경위 | 2026년 3월 말 npm 패키징 실수로 소스맵(.map) 파일에 원본 TypeScript 소스 포함 |
| 언어 | TypeScript (strict mode) |
| 런타임 | Bun (Node.js 대체) |
| UI 프레임워크 | React 19 + Ink (터미널 렌더링) |
| 전체 규모 | ~512,000줄 코드, ~2,163개 파일 |

### 1.2 Claude Code란?

Claude Code는 Anthropic이 개발한 **터미널 기반 AI 코딩 어시스턴트 CLI 도구**입니다. 단순한 채팅 인터페이스가 아니라, 실제로 코드를 읽고 수정하고 실행할 수 있는 **완전한 에이전트 시스템**입니다.

**핵심 특징:**
- 40개 이상의 에이전트 도구 (파일 읽기/쓰기, Bash 실행, 웹 검색 등)
- ~85개의 슬래시 커맨드
- 멀티 에이전트 오케스트레이션 (병렬 서브에이전트 실행)
- IDE 통합 (VS Code, JetBrains)
- MCP (Model Context Protocol) 지원
- 플러그인 및 스킬 확장 시스템
- 지속적 메모리 시스템 (CLAUDE.md)
- 웹 UI (Next.js)

## 4. 주요 설정 파일 분석

### 4.1 package.json

```json
{
  "name": "@anthropic-ai/claude-code",
  "version": "0.0.0-leaked",
  "type": "module",
  "main": "src/entrypoints/cli.tsx",
  "bin": {
    "claude": "src/entrypoints/cli.tsx"
  },
  "scripts": {
    "build": "bun scripts/build-bundle.ts",
    "build:web": "bun scripts/build-web.ts",
    "typecheck": "tsc --noEmit",
    "lint": "biome check src/",
    "format": "biome format --write src/"
  }
}
```

**핵심 포인트:**
- `"type": "module"` → ES 모듈 사용
- `bin.claude` → `claude` 커맨드가 `cli.tsx`를 실행

### 4.2 tsconfig.json 주요 설정

| 옵션 | 값 | 의미 |
|------|-----|------|
| `target` | ESNext | 최신 JS 문법으로 컴파일 |
| `module` | ESNext | ES 모듈 시스템 |
| `jsx` | react-jsx | JSX → React.createElement |
| `strict` | true | 엄격한 타입 검사 |
| `moduleResolution` | bundler | Bun 번들러에 최적화 |

### 4.3 vercel.json & server.json

- `vercel.json`: MCP 서버를 `@vercel/node`로 Vercel에 배포
- `server.json`: MCP 서버 메타데이터 (v1.1.0, "Claude Code Explorer")

## 5. 아키텍처 개요

### 5.1 전체 데이터 흐름

```
사용자 입력 (터미널)
        ↓
   [main.tsx]
   Commander.js CLI 파서
        ↓
   [screens/REPL.tsx]
   React + Ink 터미널 UI
        ↓
   [query.ts]
   쿼리 파이프라인
        ↓
   [QueryEngine.ts]
   LLM API 호출 루프
        ↓
   [Anthropic API]
   Claude 모델 (클라우드)
        ↓
   [tools/]
   도구 실행 (파일, Bash, 웹 등)
        ↓
   [components/]
   결과 렌더링
```

### 5.2 진입점 종류

| 진입점 | 파일 | 용도 |
|--------|------|------|
| CLI | `src/entrypoints/cli.tsx` | 일반 터미널 사용 |
| MCP | `src/entrypoints/mcp.ts` | MCP 서버 모드 |
| SDK | `src/entrypoints/sdk/` | 프로그래밍 API |
| Bridge | `src/bridge/bridgeMain.ts` | IDE 통합 |

### 5.3 기능 플래그 시스템

빌드 시 불필요한 코드를 완전 제거하는 `bun:bundle` feature flag:

```typescript
import { feature } from 'bun:bundle'

// 빌드 시 VOICE_MODE가 false이면 이 코드 전체 제거
if (feature('VOICE_MODE')) {
  const voiceCommand = require('./commands/voice/index.js').default
}
```

**주요 플래그 목록:**

| 플래그 | 기능 |
|--------|------|
| `PROACTIVE` | 자율 에이전트 모드 |
| `KAIROS` | 어시스턴트 모드 |
| `BRIDGE_MODE` | IDE 브릿지 |
| `DAEMON` | 백그라운드 데몬 |
| `VOICE_MODE` | 음성 입출력 |
| `AGENT_TRIGGERS` | 트리거된 에이전트 |
| `COORDINATOR_MODE` | 멀티에이전트 코디네이터 |
| `WORKFLOW_SCRIPTS` | 워크플로우 자동화 |
| `REACTIVE_COMPACT` | 반응형 컨텍스트 압축 |
| `HISTORY_SNIP` | 히스토리 스니프 압축 |
| `BG_SESSIONS` | 백그라운드 세션 |
| `TEMPLATES` | 작업 템플릿 |
| `EXPERIMENTAL_SKILL_SEARCH` | 스킬 검색 |
