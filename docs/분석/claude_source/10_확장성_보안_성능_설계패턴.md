### 1.2 플러그인 인터페이스

```typescript
export interface Plugin {
  // 필수 필드
  name: string
  version: string
  description: string

  // 도구 확장
  tools?: Array<{
    name: string
    description: string
    inputSchema: JSONSchema
    execute(input: unknown, context: ToolUseContext): Promise<{ data: string }>
    checkPermissions?(input: unknown, context: ToolUseContext): Promise<PermissionResult>
  }>

  // 커맨드 확장
  commands?: Array<{
    name: string
    description: string
    aliases?: string[]
    execute?(context: LocalCommandContext): Promise<string | JSX.Element>
    getPrompt?(args: string[]): Promise<string>
  }>

  // 시스템 프롬프트 확장
  systemPromptAddendum?: string

  // 훅
  hooks?: {
    beforeQuery?: (messages: Message[]) => Promise<Message[]>
    afterQuery?: (messages: Message[]) => Promise<void>
    onToolCall?: (toolName: string, input: unknown) => Promise<void>
    onToolResult?: (toolName: string, result: string) => Promise<void>
  }

  // 라이프사이클
  onInstall?: () => Promise<void>
  onEnable?: () => Promise<void>
  onDisable?: () => Promise<void>
  onUninstall?: () => Promise<void>
}
```

### 1.3 플러그인 개발 예시

```typescript
// ~/.claude/plugins/my-plugin/index.ts
import type { Plugin } from '@anthropic-ai/claude-code'

const myPlugin: Plugin = {
  name: 'my-awesome-plugin',
  version: '1.0.0',
  description: '내 커스텀 플러그인',

  tools: [
    {
      name: 'DatabaseQuery',
      description: '내부 데이터베이스 쿼리 실행',
      inputSchema: {
        type: 'object',
        properties: {
          query: { type: 'string', description: 'SQL 쿼리' }
        },
        required: ['query']
      },
      async execute({ query }) {
        const result = await db.execute(query)
        return { data: JSON.stringify(result, null, 2) }
      }
    }
  ],

  commands: [
    {
      name: 'db-status',
      description: '데이터베이스 연결 상태 확인',
      async execute() {
        const status = await db.ping()
        return `DB 상태: ${status ? '✓ 연결됨' : '✗ 연결 실패'}`
      }
    }
  ],

  systemPromptAddendum: `
You have access to an internal database via the DatabaseQuery tool.
Always prefer using DatabaseQuery for data retrieval instead of file operations.
  `,
}

export default myPlugin
```

### 2.4 스킬 로더

```typescript
export async function loadAllSkills(): Promise<Skill[]> {
  const skillDirs = [
    // 내장 스킬
    path.join(__dirname, 'bundled'),

    // 글로벌 스킬
    path.join(os.homedir(), '.claude', 'skills'),

    // 프로젝트 스킬
    path.join(getCwd(), '.claude', 'skills'),
  ]

  const skills: Skill[] = []

  for (const dir of skillDirs) {
    if (!existsSync(dir)) continue

    const entries = readdirSync(dir)
    for (const entry of entries) {
      const skillFile = path.join(dir, entry, 'skill.yaml')
      const markdownFile = path.join(dir, `${entry}.md`)

      if (existsSync(skillFile)) {
        skills.push(await loadYamlSkill(skillFile))
      } else if (existsSync(markdownFile)) {
        skills.push(await loadMarkdownSkill(markdownFile))
      }
    }
  }

  return skills
}
```

## 3. 메모리 시스템 (src/memdir/)

### 3.1 메모리 계층 구조

Claude Code의 메모리는 **파일 기반 지속성 시스템**입니다:

```
메모리 계층:
┌─────────────────────────────────┐
│ 글로벌 메모리                     │
│ ~/.claude/CLAUDE.md             │
│ ~/.claude/memory/*.md            │
├─────────────────────────────────┤
│ 프로젝트 메모리                   │
│ .claude/CLAUDE.md (프로젝트 루트) │
│ .claude/memory/*.md              │
├─────────────────────────────────┤
│ 로컬 메모리 (gitignore됨)         │
│ .claude/CLAUDE.local.md          │
└─────────────────────────────────┘
```

### 3.2 CLAUDE.md 자동 발견

```typescript
// 현재 디렉토리에서 상위로 탐색하며 CLAUDE.md 파일 수집
export async function findAndLoadClaudeMd(cwd: string): Promise<string> {
  const files: string[] = []
  let dir = cwd

  while (true) {
    const claudeMd = path.join(dir, 'CLAUDE.md')
    const claudeLocalMd = path.join(dir, 'CLAUDE.local.md')

    if (existsSync(claudeMd)) {
      files.push(claudeMd)
    }
    if (existsSync(claudeLocalMd)) {
      files.push(claudeLocalMd)  // 항상 gitignore
    }

    // 루트 디렉토리에 도달하면 중지
    const parent = path.dirname(dir)
    if (parent === dir) break
    dir = parent
  }

  // 글로벌 CLAUDE.md
  const globalClaudeMd = path.join(os.homedir(), '.claude', 'CLAUDE.md')
  if (existsSync(globalClaudeMd)) {
    files.push(globalClaudeMd)
  }

  // 모든 파일 내용 결합
  return files
    .map(f => readFileSync(f, 'utf-8'))
    .join('\n\n')
}
```

### 3.3 메모리 시스템 컨텍스트 (auto memory)

```typescript
// 메모리 파일들을 시스템 프롬프트에 자동 포함
export function buildMemorySystemPrompt(memoryDir: string): string {
  const memoryIndex = path.join(memoryDir, 'MEMORY.md')

  if (!existsSync(memoryIndex)) {
    return ''
  }

  const indexContent = readFileSync(memoryIndex, 'utf-8')

  // MEMORY.md의 링크를 파싱하여 개별 메모리 파일 로드
  const linkedFiles = parseMemoryLinks(indexContent)
  const memories = linkedFiles.map(file => {
    const content = readFileSync(file, 'utf-8')
    return parseFrontmatter(content)
  })

  return formatMemoriesForPrompt(memories)
}
```

## 4. 권한 시스템 상세

### 4.1 권한 모드

```typescript
type PermissionMode =
  | 'default'              // 각 위험한 작업마다 사용자 확인
  | 'acceptEdits'          // 파일 편집은 자동 승인, Bash는 확인
  | 'plan'                 // 먼저 계획 표시, 한 번만 승인
  | 'bypassPermissions'    // 모든 작업 자동 승인 (위험!)
  | 'auto'                 // ML 기반 자동 판단 (실험적)
```

### 4.2 권한 규칙 문법

```
// 권한 규칙 형식
<ToolName>(<pattern>)

// 예시
Bash(git *)              → git으로 시작하는 모든 bash 명령
Bash(npm test)           → 정확히 "npm test"만
Bash(npm run *)          → "npm run ..."
FileEdit(src/*)          → src/ 하위 파일 편집
FileRead(*)              → 모든 파일 읽기
WebFetch(https://*)      → https URL 가져오기

// 와일드카드
*                        → 모든 것 허용
Bash(*)                  → 모든 Bash 명령 허용
```

## 6. 핵심 설계 패턴

### 6.1 에이전트 루프 패턴

Claude Code의 핵심 패턴: **반복적 도구 호출**

```
Input
  ↓
LLM Call
  ↓
Response 분석
  │
  ├─ text: 출력 & 계속
  ├─ tool_use: 도구 실행 → 결과 추가 → LLM 재호출
  └─ end_turn: 완료
```

이 패턴이 "에이전트"의 본질: LLM이 스스로 도구를 선택하고 순서를 결정하여 복잡한 작업을 수행.

### 6.2 Generator 기반 스트리밍

// AsyncGenerator로 메시지 스트림 처리

async function* processAgentStream(
  agentStream: AsyncGenerator<Message>

  }
}

## 9. 전체 시스템 요약

### 9.1 핵심 아키텍처 원칙

1. **도구 기반** - 모든 기능이 도구로 캡슐화
2. **에이전트 루프** - LLM이 자율적으로 도구 선택 및 실행
3. **다층 보안** - 여러 단계의 검증과 권한 확인
4. **확장 가능** - 플러그인, 스킬, MCP로 런타임 확장
5. **성능 최적화** - 병렬 로딩, 캐싱, 컴파일 최적화
6. **격리** - 에이전트 간, Worktree 격리
7. **관찰 가능성** - 비용 추적, 텔레메트리, 진행률 표시

### 9.2 완전한 실행 흐름

```
사용자 입력
    ↓
main.tsx (CLI 파싱, 초기화)
    ↓
screens/REPL.tsx (React/Ink UI)
    ↓
query.ts (파이프라인, 컨텍스트 조립)
    ↓
QueryEngine.ts (LLM 루프)
    ↓
Anthropic API (Claude 모델)
    ↓
도구 실행 루프:
  BashTool → 보안 → 권한 → 실행
  FileEditTool → 유일성 검증 → 쓰기
  AgentTool → 서브에이전트 생성 → 병렬 실행
    ↓
결과 렌더링 (Ink 컴포넌트)
    ↓
비용 추적 + 분석 로깅
    ↓
메시지 히스토리 저장
```
