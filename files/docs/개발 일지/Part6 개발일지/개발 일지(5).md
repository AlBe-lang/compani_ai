## 3. 결정 이유

### 3-1. 왜 `models.Document` inline inference인가 (vs 명시 TextEmbedding)
대안:
- **(A) `models.Document(text, model=...)` 인라인**: Qdrant가 fastembed를 내부 호출하여 임베딩. 코드 단순, 모델 변경 시 1상수만 수정
- **(B) 명시 `TextEmbedding(model_name)` + `PointStruct(vector=raw_vec)`**: 직접 임베딩 후 raw vector 전달. 모델 라이프사이클 통제 가능, 더 verbose

A 채택 — 다음 이유:
1. **Rule 9.1 P-04 회피**: B는 EmbeddingProvider 추상화 도입의 첫 단계처럼 보일 수 있어 합본 정비 범위를 벗어남. 사용자 합의(Q3): 추상화는 Phase 8 또는 별도 Stage로 분리
2. **호출 비용 동일**: A도 fastembed를 호출하므로 성능 차이 없음 (테스트 평균 +0.3s/query 정도, 모델 first-load 비용 외)
3. **변경 가독성**: 한 줄 상수 변경(`_EMBEDDING_MODEL`)으로 모델 교체 가능 — Phase 8에서 e5-large로 갈아끼울 때도 동일 패턴

### 3-2. 왜 `paraphrase-multilingual-MiniLM-L12-v2` 모델인가
fastembed 0.7.4가 지원하는 다국어 모델:
| 모델 | 디멘션 | 크기 | 언어 | 비고 |
|------|--------|------|------|------|
| paraphrase-multilingual-MiniLM-L12-v2 | 384 | ~117MB | ~50 (ko 포함) | **채택** |
| paraphrase-multilingual-mpnet-base-v2 | 768 | ~420MB | ~50 | 정확도 ↑, 메모리 ↑ |
| intfloat/multilingual-e5-large | 1024 | **2.24GB** | ~100 | 최강 정확도, 메모리 부담 |

PROJECT_PLAN line 44: **Mac Mini M4 16GB** 하드웨어 제약. CTO `llama3.1:8b`(~4.7GB) + SLM `gemma4:e4b`(~9.6GB)가 동시 로드되는 환경에서 임베딩 모델은 **117MB의 MiniLM이 가장 안전**. 검증 결과 한국어 쿼리가 한국어 레코드를 정확히 매칭(테스트 통과)하므로 정확도도 routing 용도로는 충분.

E5-large는 Phase 8 통합 최적화에서 정확도 요구가 더 커지면 재고려.

### 3-3. 왜 컬렉션을 `init()`에서 사전 생성하는가
구 API(`client.add()`)는 첫 쓰기 시 fastembed가 named-vector 구조로 자동 생성했지만, 신 API(`client.upsert(points=[PointStruct(vector=Document(...))])`)는 **컬렉션이 미리 존재해야 함**. 자동 생성 흐름을 잃었으므로 `_ensure_collections()` 헬퍼로 명시 처리. 멱등성 보장(`get_collections()`로 존재 확인 후 미생성 시만 생성)으로 `init()` 반복 호출 안전.

### 3-4. 왜 `D-05 회피` 방식으로 일지(4) 수정 대신 새 일지에 안내문을 넣었는가
Rule 8.2 D-05: "기존 개발 일지 파일 소급 수정 금지". 일지(4) 본문을 직접 고치면 위반. 그러나 정정이 명시되지 않으면 Part 7 진입자가 또 혼란을 겪음. 절충: 신규 일지(본 문서)에 정정 안내를 명시하고, 정본 위치(PROJECT_PLAN §8)와 메타 파일 동기화 위치(`세션 작업 가이드.md`)를 분명히 가리킴. 이는 D-05의 정신(역사 보존)을 지키면서 운영상 혼란을 차단.

### 3-5. 왜 `세션 작업 가이드.md`는 직접 수정했는가
세션 작업 가이드는 **개발 일지가 아닌 메타 가이드 파일**이므로 D-05 적용 대상이 아니다. 또한 다음 세션 시작 시 Step 1 컨텍스트 주입에 직접 사용되어, 정정이 반영되지 않으면 즉시 영향. 사용자 직접 지시("일치하도록 갱신하세요")로 본 합본 정비 시작 직전에 별도 커밋 단위로 처리됨.
