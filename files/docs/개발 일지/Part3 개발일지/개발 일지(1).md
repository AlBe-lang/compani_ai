## 3. 결정 이유

- **`framework` 필드 strip**: `TaskResult(extra="forbid")`에서 unknown 필드 거부 → domain 모델 변경 없이 application 레이어에서 처리. 이미 `_validate_files`에서 검증된 후라 strip해도 정보 손실 없음
- **React/Flutter/Test 트랙 분리**: Stage 4는 Flutter로 완전히 다른 기술 스택이므로 Stage 1-3 누적 검증을 Stage 4에 적용할 수 없음. 트랙 분리로 각 프레임워크 독립 검증
- **`_REACT_STAGES`, `_FLUTTER_STAGES`, `_TEST_STAGES` 상수**: 검증 분기 조건을 명시적으로 관리, 향후 Stage 추가 시 집합에 번호만 추가
- **전략 변경 없음**
