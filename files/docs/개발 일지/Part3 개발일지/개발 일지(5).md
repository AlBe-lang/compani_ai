## 3. 결정 이유

- **Riverpod 강제**: 규칙 문서 Flutter 상태 관리 기준. `StatefulWidget + setState` 만으로는 불충분, 별도 provider 레이어 필수
- **Riverpod 키워드 검증**: `"flutter_riverpod"` 패키지명이 코드에 포함되어야 함. provider 파일 경로 + 패키지 import 두 가지 검증으로 실제 사용 여부 확인
- **HTTP 호출 패턴 (`http.get(` 또는 `http.post(`)**: `http` 단어만으로는 import 문에서도 매칭되므로, `.get(` 또는 `.post(` 추가 조건으로 실제 호출 여부 확인
- **전략 변경 없음**
