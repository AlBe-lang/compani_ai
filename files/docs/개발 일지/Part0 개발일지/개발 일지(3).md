# Part0 개발 일지 (3) — Stage 3: 도메인 계약 정의 (스키마 고정)
> 작성일: 2026-02-21 | Part: 0 | Stage: 3

---

## 1. 작업 내용
Stage 3에서 `domain/contracts` 패키지를 신설하고 `WorkItem`, `Message`, `Strategy/Task`, `TaskResult/FileInfo`, `AgentDNA` 계약 모델을 Pydantic으로 고정했습니다.  
기존 `domain/ports.py`의 `dict` 기반 시그니처를 계약 타입(`WorkItem`, `Message`, `TaskResult`, `WorkStatus`, `MessageType`)으로 교체해 Stage 1에서 남겨둔 타입 부채를 정리했습니다.  
Mock 어댑터(`mock_workspace`, `mock_message_queue`)와 스모크 테스트를 새 계약에 맞게 리팩터링하고, 각 계약별 직렬화/역직렬화 단위 테스트를 추가했습니다.

## 2. 결과물
생성된 파일: `src/domain/contracts/__init__.py`, `src/domain/contracts/work_item.py`, `src/domain/contracts/message.py`, `src/domain/contracts/strategy.py`, `src/domain/contracts/task_result.py`, `src/domain/contracts/agent_dna.py` — 도메인 계약 모델.  
수정된 파일: `src/domain/ports.py`, `src/domain/__init__.py`, `src/adapters/mock_workspace.py`, `src/adapters/mock_message_queue.py`, `src/adapters/mock_llm_provider.py`, `tests/unit/test_stage1_smoke.py` — 계약 타입 정렬 및 회귀 대응.  
생성된 테스트: `tests/unit/test_contract_work_item.py`, `tests/unit/test_contract_message.py`, `tests/unit/test_contract_strategy.py`, `tests/unit/test_contract_task_result.py`, `tests/unit/test_contract_agent_dna.py`.

## 3. 결정 이유
도메인 계약을 `src/domain/contracts`에 집중 배치해 모든 Part가 동일한 SSOT 스키마를 참조하도록 만들었습니다.  
`ports`의 `dict[str, ...]` 대신 계약 모델을 직접 타입으로 사용해 이후 Part 1~5에서 파싱/변환 비용과 타입 모호성을 줄이고 리팩터링 비용을 낮췄습니다.  
전략 문서(개발 일지(4).md) Part 0 Stage 3의 "스키마 고정 + 테스트" 목표에 맞춰 구현했으며, 전략 변경 이력은 없음입니다.
