> 작성일: 2026-02-21 | Part: 0 | Stage: 4

## 1. 작업 내용
Stage 4에서는 GitHub Actions CI 워크플로우를 추가해 lint/unit/integration 3개 Job을 분리 구성했습니다.  
실행 스크립트 `deploy/run.sh`를 추가해 Ollama CLI/데몬/모델 존재 확인 후 앱을 실행하도록 구성했습니다.  
`main.py` 비동기 스켈레톤과 `docs/CONTRIBUTING.md`(Conventional Commit + BREAKING CHANGE 규칙)를 작성해 Part 0 마무리 기반을 갖췄습니다.

## 2. 결과물
생성된 파일: `.github/workflows/ci.yml` — lint, unit, integration 자동 검증 파이프라인.  
생성된 파일: `deploy/run.sh` — `set -euo pipefail`, Ollama 사전 점검, 모델 자동 pull, 앱 실행 스크립트.  
생성된 파일: `main.py` — 비동기 엔트리포인트 스켈레톤.  
생성된 파일: `docs/CONTRIBUTING.md` — 커밋 규칙 및 BREAKING CHANGE 정책 문서.

## 3. 결정 이유
CI는 Stage 4 목표에 맞춰 E2E(Ollama 의존)를 제외하고 정적품질+단위/통합 테스트만 자동화했습니다.  
`deploy/run.sh`는 실패를 빠르게 드러내기 위해 Ollama 미설치/미기동 시 즉시 `[ERROR]`로 종료하도록 작성했습니다.  
전략 문서(개발 일지(4).md) Part 0 Stage 4 기준을 따랐고, 전략 변경 이력은 `main.py`를 최소 스켈레톤으로 유지한 점(Part 5 배선 예정)입니다.

## 4. 발생한 리스크
리스크: 로컬 환경에서는 `black/flake8/isort` 미설치로 lint를 즉시 실행하지 못할 수 있음 | 심각도: B | 발현 시점: 로컬 `make lint` 실행 시 | 대응: `requirements-dev.txt` 설치 후 lint 게이트 확인.  
리스크: CI는 아직 실제 GitHub 원격 PR 실행/green merge 검증 전 상태 | 심각도: B | 발현 시점: Part 0 Gate 최종 확인 시 | 대응: 원격에 브랜치 푸시 후 첫 PR로 CI green 여부 검증.
