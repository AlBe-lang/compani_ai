## 2. 기술적 결정 사항

### A. `_AUTH_KEYWORDS`에서 `"security"` 제외
**이유**: `from backend.security import get_current_user` 같은 임포트 경로에서 false positive 발생.  
`bearer`, `oauth2`, `jwt`는 실제 인증 구현 코드에서만 등장하는 기술 특정 키워드로 제한.

### B. 누적 검증 패턴 유지
Stage 4는 Stage 2 + Stage 3 요건을 모두 포함해야 함.  
`_validate_files()` 호출 체인: base → stage2 → stage3 → stage4.  
테스트 데이터의 `_base_files()`에 이전 단계 필수 파일 모두 포함.
