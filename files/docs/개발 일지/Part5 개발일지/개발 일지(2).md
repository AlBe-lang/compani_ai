## `SharedWorkspace` 구현

### 이중 레이어 구조
```
SharedWorkspace
├── _cache: dict[str, WorkItem]   ← 인메모리 캐시 (빠른 읽기)
├── _storage: StoragePort         ← SQLiteStorage (영속화)
└── _event_bus: EventBusPort      ← 상태 변경 이벤트 발행
```

`get()` 시 캐시 우선 조회 → 미스 시 DB에서 로드 후 캐시 갱신.

### 상태 전이 규칙
```python
_ALLOWED_TRANSITIONS = frozenset({
    (PLANNED, IN_PROGRESS),
    (PLANNED, WAITING),
    (WAITING, IN_PROGRESS),
    (IN_PROGRESS, DONE),
    (IN_PROGRESS, FAILED),
    (IN_PROGRESS, BLOCKED),
    (BLOCKED, IN_PROGRESS),
})
```
허용되지 않은 전이 시 `AdapterError(E_DEPS_BLOCKED)` 발생.

### 이벤트 발행
`set_status()` 호출마다 `"work_item.updated"` 이벤트 발행:
```python
{"item_id": ..., "prev_status": ..., "curr_status": ...}
```

## `SQLiteMessageQueue` 구현

### asyncio + SQLite 조합
- **asyncio.Queue**: 실시간 전달 (동일 프로세스 내)
- **SQLiteStorage**: 히스토리 영구 보존

```
send() → SQLite 저장 + asyncio.Queue 투입
receive() → asyncio.Queue에서 꺼내기
ask() → QUESTION 발송 + from_agent 인박스에서 ANSWER 대기 (타임아웃)
```

### 타임아웃 처리
`ask()` 타임아웃 시 에러 없이 `""` 반환 — 규칙 C-06 준수.
