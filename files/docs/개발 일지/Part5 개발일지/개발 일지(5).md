## 결정 이유

### 왜 E2E 테스트에서 실제 Ollama를 사용했는가?

Mock 기반 Gate는 실제 LLM 연동 버그를 잡지 못한다. 이전 세션에서 사용자가 "Gate에서 Mock 쓰면 안 됨, 실제 Ollama 연결"을 명확히 요구. `@pytest.mark.slow` 로 분리하여 CI 기본 실행에서는 제외하되 릴리즈 전 실행하도록 설계.

### 왜 integration test 에 dependencies=[] 태스크만 사용했는가?

`BaseSLMAgent._wait_dependencies` 는 `task.dependencies` (task ID) 로 `workspace.get()` 을 호출하는데, workspace는 `work_item_id`(= `work_{task_id}_{uuid}`) 로 저장한다. 즉, 의존성 대기 시 항상 `None` 리턴 → `E_DEPS_BLOCKED`. 이 구조적 불일치는 기존 코드의 알려진 제약. 통합 테스트에서는 의존성 없는 태스크만 사용하여 이 경로를 회피. 수정은 Part 6 이후 별도 이슈로 추적.
