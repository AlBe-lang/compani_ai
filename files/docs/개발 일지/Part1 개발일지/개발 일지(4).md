## 1. 작업 내용
Stage 4 목표에 맞춰 `CTOAgent`의 관측성과 파싱 복원력을 강화했습니다.
삭제되어 있던 `src/observability/parsers.py`를 복구하고, 코드블록(````json`, `````) 추출 + 부분 JSON 복구 시도를 추가했습니다.
`create_strategy`, `decompose_tasks`, `review_progress` 성공 로그에 `response_chars`를 포함하도록 수정했습니다.
`strategy.txt`, `decompose.txt`에 few-shot 예시를 추가해 LLM 출력 스키마 일관성을 높였습니다.

## 3. 결정 이유
개발 일지(4) Part 1 Stage 4 DoD의 핵심 요구(부분 JSON 복구, few-shot 예시, 토큰 로그)를 우선 충족하도록 범위를 고정했습니다.
`json_repair`는 설치 유무가 환경마다 다를 수 있어 optional import + 내부 fallback 복구를 함께 두어 실행 안정성을 확보했습니다.
`response_chars`는 응답 원문 길이 기반 지표로 정의해 의미가 모호하지 않도록 했습니다.
기존 Stage 1~3 테스트 회귀를 유지하면서 Stage 4 신규 동작을 단위 테스트로 분리했습니다.
