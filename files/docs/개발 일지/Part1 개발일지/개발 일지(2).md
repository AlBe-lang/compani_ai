# Part1 개발 일지 (2) — Stage 2: 작업 분해 (decompose_tasks)
> 작성일: 2026-02-22 | Part: 1 | Stage: 2

---

## 1. 작업 내용
이번 Stage에서는 `CTOAgent.decompose_tasks(strategy)`를 구현해 Strategy를 Task 목록으로 분해하고, 의존성 검증 및 실행 순서 정렬까지 포함했습니다.  
지시된 리뷰 항목을 반영해 `CTOAgent.__init__` 시그니처를 `llm + workspace + team + config + run_id` 형태로 유지했고, decompose LLM 호출 타임아웃은 60초로 분리 적용했습니다.  
분해 결과에 대해 Task 개수 제한(최소 3, 최대 15), 의존성 참조 유효성, 순환 의존성 검증을 추가하고 위상 정렬 결과를 반환하도록 구성했습니다.

## 2. 결과물
수정된 파일: `src/application/cto_agent.py` — `decompose_tasks`, 의존성 검증, 위상 정렬, decompose 프롬프트 로딩, config에 `decompose_timeout_sec` 추가.  
생성된 파일: `src/application/prompts/cto/decompose.txt` — Stage 2 분해 출력 스키마 및 제약 규칙.  
생성된 테스트: `tests/unit/test_cto_decompose.py` — 정상 분해/순환 의존성/잘못된 의존성/Task 개수 제한 검증.  
검증 결과: `python -m pytest tests -m "not slow" -q` 30 passed, `python -m mypy src tests main.py` 통과.

## 3. 결정 이유
개발 일지(4) Part 1 Stage 2 DoD의 "의존성 없는 Task 먼저 배분"을 충족하기 위해 LLM 반환 순서가 아니라 위상 정렬 결과를 표준 반환으로 채택했습니다.  
규칙 문서(01_ORCHESTRATION_RULES 2절)를 따라 전략 수립(120초)과 작업 분해(60초) 타임아웃을 분리했고, 재시도 간격은 동일하게 2/4/8초를 유지했습니다.  
전략 변경 이력은 없으며, 이전 리뷰의 A급 지적(생성자 시그니처)을 Stage 2 구현에 포함해 선제 해소했습니다.
